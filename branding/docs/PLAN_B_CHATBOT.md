# Plan B — bapmap.com 카카오 챗봇 파이프라인

> 목표: bapmap.com 맛집 데이터를 기반으로 카카오톡 채널 챗봇을 구축, 사용자가 대화로 맛집을 추천받을 수 있는 서비스

---

## 1. 서비스 개요

| 항목 | 내용 |
|------|------|
| 플랫폼 | 카카오톡 채널 (카카오 i 오픈빌더) |
| 타겟 | 한국 여행 계획 중인 내/외국인 |
| 핵심 가치 | 대화로 맛집 추천 — 지역, 음식 종류, 분위기, 예산 맞춤 |
| bapmap 연동 | 추천 결과 → bapmap 상세 페이지 링크 제공 |

---

## 2. 챗봇 기능 정의

### 핵심 기능
| 기능 | 설명 |
|------|------|
| 맛집 추천 | 지역 + 음식 종류 + 예산 + 분위기 기반 추천 |
| 지역 가이드 | "명동 맛집 알려줘" → 해당 지역 큐레이션 |
| 테마 추천 | "혼밥", "야식", "데이트" 등 상황별 추천 |
| 식당 상세 | 특정 식당 정보 조회 (위치, 메뉴, 가격) |
| 블로그 연결 | 관련 블로그 포스트 링크 제공 (Plan A 연동) |

### 대화 플로우 예시
```
사용자: 강남에서 고기 먹고 싶어
챗봇: 예산이 어떻게 돼요?
  [2만원대] [3만원대] [상관없어요]

사용자: [3만원대]
챗봇: 강남 고기집 추천 3곳이에요 🥩
  1. 불이아 역삼점 ⭐4.2 - 역삼역 3분
  2. ...
  [자세히 보기 → bapmap.com/...]
```

---

## 3. 파이프라인 단계

```
[STEP 1] 사용자 입력 수신
    - 카카오 i 오픈빌더 → Webhook → 서버

[STEP 2] 의도 파악 (Intent Detection)
    - LLM으로 사용자 의도 분류
    - 지역 / 음식종류 / 예산 / 분위기 추출 (NER)

[STEP 3] bapmap DB 검색
    - 조건에 맞는 식당 필터링
    - 평점 + 관련도 기준 정렬

[STEP 4] 응답 생성 (LLM)
    - 자연스러운 추천 문구 생성
    - 카카오 말풍선 포맷 (리스트형, 캐러셀형)

[STEP 5] 응답 전송
    - 카카오 i 오픈빌더 응답 포맷으로 변환
    - 식당 카드 + bapmap 링크 + 블로그 링크 포함

[STEP 6] 대화 로그 저장
    - 사용자 질의 패턴 DB 저장
    - 많이 물어보는 지역/음식 분석

[STEP 7] 개선 루프
    - 자주 묻는 질문 → FAQ 카드 추가
    - 미응답 케이스 → 프롬프트 개선
    - 리포트 생성
```

---

## 4. 기술 스택

| 컴포넌트 | 도구 |
|----------|------|
| 챗봇 빌더 | 카카오 i 오픈빌더 |
| 백엔드 서버 | FastAPI (Python) |
| 의도 파악 / 응답 생성 | Claude API (claude-sonnet-4-6) |
| DB | Supabase (Plan A, C와 공유) |
| 배포 | Railway or Fly.io |
| 로깅 | Supabase 대화 로그 테이블 |

## 7. Phase별 구현 계획

### Phase 1 — 수동 워크플로우 검증
- [ ] 카카오 i 오픈빌더 채널 개설

### Phase 2 — 백엔드 연동
- [ ] FastAPI 서버 구축
- [ ] 카카오 Webhook 연동
- [ ] Claude API 의도 파악 + 응답 생성
- [ ] Supabase DB 연결
- [ ] 배포 (Railway)

- [ ] 멀티턴 대화 관리 (대화 컨텍스트 유지)
